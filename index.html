WEB SERVERS

In web development, index is a standard naming convention for a website's home page. When a user goes to any website, such as www.google.com, the browser renders contents of index.html from one of Google's servers. The user will type the domain name in the browser's address bar and hit enter. The browser will send a GET request to the IP address that the domain name points to, which is the server hosting the Google index page. When the server sees the request, it will respond with a status code, such as 200, and provide the contents of index.html to the browser. After receiving the response from the server, the browser renders the HTML content and shows the user Google's home page.

Users can open Web Dev / Inspector tools in their respective browsers to better understand this process. In Chrome, this can be accomplished by going to View, Developer, and then Developer Tools. Developers can directly observe the network activity described above by clicking the Network tab. Users should note that the browser network activity is slightly different when viewing a page directly from the local computer (by dragging and dropping an HTML page into the browser) and when the browser requests the same page hosted by a server. If a user pulls an HTML page into Google Chrome, they should note that the page URL will start with file:// instead of http:// or https://. The browser is not actually making a request to a remote server; it's asking the operating system to fetch the file the user dragged from the local file system. Only three items are visible in the network tab when making this local request: index.html (200), main.css (failed), and main.js (failed). "Failed" means that the operating system did not find main.css and main.js in this case. On the hand, looking at the network tab of this page when it has been uploaded to a remote server will reveal a few slight differences. The URL starts with https://, which means that the browser made a GET request to a remote server this time. Instead of seeing index.html, the user will see a1-gevkob/, the root directory where the index file is located. Additionally, main.css and main.js have 404 status codes instead of failed, signifying that the server could not find these files. Finally, the server provided two additional responses: single-file-hooks-frames.js (200), a JavaScript file, and favicon.ico (200), an icon.

In addition to using Developer Tools, the developer can gain valuable insights from the URL of a given web page. The URL for this page (https://web.engr.oregonstate.edu/~gevkob/a1-gevkob/) can be broken down as follows: The first segment of the URL, https, tells the developer that the browser will make a request to the server using the secure hypertext transfer protocol, and it is also known as the scheme. The next segment, web, is known as the subdomain. The central part of the URL is engr.oregonstate.edu, which is the host domain. And lastly, the /~gevkob/a1-gevkob/ aspect of the URL represents the resources of the URL, which is the specific location where the index.html file is located for this page.

FRONTEND DESIGN

Frontend in web development refers to the aspect of a website that the user sees and interacts with. Designing the frontend includes making decisions regarding the visual elements of a website, such as layout, fonts, colors, and images. Additionally, frontend design focuses on the user's experience or how the overall feel of the website. There are many tools to assist developers with frontend design, but on the most fundamental level, developers use HTML, CSS, and JavaScript to design the look and feel of a website.

When designing effective websites, developers think of the user experience in terms of usability. Usability is the ease with which the user is able to use the website and is broken down into five sections, also known as the Five "E"s of usability:

Effective
The ease with which a user can complete a task using the site, such as purchasing a product or finding information.
Efficient
Refers to how quickly the user can accomplish a given task. An efficient site will enable users to achieve their desired tasks without friction.
Engaging
An engaging site will feel pleasant to the user. Details like a pleasant art style or visual feedback when hovering over an element can make a site more engaging.
Error Tolerant
Developers can make a site error tolerant by handling and anticipating common user errors. For example, if a user enters an incorrect password, the website should provide feedback and allow the user to correct their mistake.
Easy to Learn
Users are able to intuitively get the site. They can immediately start using the website without needing a tutorial.

When designing the page layout, it's important to note that although some HTML tags have similar behavior, they have different semantic purposes. The developer could, in theory, use the general-purpose <div> tag for every aspect of the page, but doing so severely hinders usability for blind users who rely on screen readers. Instead, the developer should follow the standardized semantic rules and use appropriate tags. The <header> tag represents the website's top banner, which usually contains the website name, publisher, and marketing slogan. The <nav> tag includes navigation links that take users to other parts of the site. The <main> element goes inside the <body> tag and represents the primary block containing content, such as text, images, and videos. The <section> tag is usually one of the children of <main> and is used to group related content; it uses the <h1> headline to describe what the section is about. The <article> tag is used inside <section> elements and denotes a specific topic defined by the second-level headline <h2>. The developer can include extra information related to the content in an <article> tag by using the <aside> element, which typically floats to the left or right of that article. The <aside> element is also sometimes used to create sidebars for navigation. The <figure> tag usually wraps the <img> and the <figcaption> tags to provide additional information to the user in the form of images or video. The developer can also use the <blockquote> tags to help paragraphs stand out, which can also include the quote and the citation tag, or <q> and <cite>, respectively. Lastly, the <footer> tag is generally found at the bottom of the page and may contain legal information, contact information, and links to other pages.

HTML also provides various types of navigation functionality, depending on the content:

  1. External Content: The <a> tag is used with the href attribute to specify the URL of an external resource. <a href="https://example.com">Example</a> would create a link called Example. 
  2. Internal Content: Anchors can be used to link to specific sections within the same HTML page by including the id attribute inside the href. <a href="#section1"> Go to Section 1 </a> would create a link titled "Go to Section 1", assuming that another element has the id attribute set to "#section1."
  3. Page-to-Page: The anchor tag can also link to another HTML page in the specified directory. <a href="about.html">About</a> would lead the user to the About page if about.html exists in the same directory as index.html.

OPTIMIZED IMAGES

Images can add a lot to a website. Although adding images to a website seems straightforward, web developers should follow best practices to ensure the site loads quickly. Image file names should be concise and descriptive to improve search engine optimization (SEO). Image files should be small without visibly affecting the image quality, which can be achieved through various compression tools. Developers should also crop their images to ensure the image fits within the page and that no space is wasted. Furthermore, developers should note that the various image file formats have different purposes. GIF files support binary transparency (either fully transparent or fully opaque) and are commonly used for short animations. GIF supports 256 colors and offers lossless compression, which is also ideal for line art. PNG files support 24-bit color depth (16.7 million colors) and full transparency through an alpha channel and are most suitable for images with complex shapes, lines, colors, and overlapping elements. SVG files, also known as vector graphics, are mathematically derived images that do not distort when resized to an extreme degree and are often used for logos, icons, interactive images, and animations. JPG files are the best format for photographs and can be significantly compressed without noticeably losing image quality. Lastly, WebP files are also used for photographs, but unlike JPG files, WebP images support transparency through an alpha channel. JPG, WebP, and PNG files all support 24-bit color depth. However, unlike the PNG format, which only supports lossless compression, JPG and WebP formats offer lossy compression, which can significantly reduce file sizes (often without noticeable impact), making them ideal for photographs.

FAVICONS

A favicon, short for "favorites icon," is a category of images in web development that come in the form of small icons and are used to quickly identify a site in various contexts. Favicons represent a website or app logo and can be displayed in the browser tab, browser bookmarks, address bar, mobile devices, and generally other places where icons are expected. The most common file types for favicons are SVG and PNG, but formats such as GIF, PNG, and ICO are also supported. These files go in the root directory of a website, with each copy having a different size. The varying dimensions provide icons for browser and high-resolution device use; common sizes are 16x16, 32x32, 180x180, 192x192, and 512x512 pixels. Note that the 180x180 size is reserved for Apple mobile devices, and the other two bigger dimensions will be used by an Android device. Once all the sizes are set, the developer can set a reference to these icons in the <head> tag, and the browser will automatically show them in the correct context. This can be done with the following references: 

<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
<link rel="manifest" href="/site.webmanifest">

The first link tells the browser which icon should be used if users add a website to their home screen on their Apple mobile device. The next two lines tell the browser where to find icons for regular web use; the bigger icon is used unless space is limited. The last link refers to a web manifest file (site.webmanifest), which may look something like this: 


{
  "name": "Your App Name",
  "short_name": "App",
  "icons": [
    {
      "src": "/android-chrome-192x192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/android-chrome-512x512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ],
    "theme_color": "#ffffff",
    "background_color": "#ffffff",
    "display": "standalone"
}

The site manifest file, also in the website directory's root folder, will tell the browser which icon to use when a user saves a website to the home screen of their Android mobile device. The site manifest file has other uses as well, but that's outside the scope of this article.

CASCADING STYLE SHEETS

Cascading Style Sheets (CSS) are at the core of how a website looks and feels. CSS gives the developer the ultimate control over every element of a website. Standard HTML comes with default styles that are rarely used in modern web development. To have creative control over a website's style, clarity, and accessibility, the developer has to turn to CSS. CSS can tell the browser how to render content in several ways:

Inline
Inline styles are written directly into HTML elements using the style attribute. This is a good method for quick touchups, but it can quickly become tedious for more complex designs. 
<p style="color: blue;">Hello, I'm blue..</p>
Embedded
Embedded styles are defined within the HTML document using the <style> element. This way of working is ideal for web pages that don't require shared styles. 
  <head>
    <style>
      p {
        color: blue;
      }
    </style>
  </head>
JavaScript
Turning to JavaScript for CSS styling is best suited for situations where the developer needs dynamic styling, such as when a user clicks an element or scrolls down. The JavaScript that applies the given style can be embedded into HTML or written in an external file. Here's an example that demonstrates adding a border to an
element: 
<body>
  <nav>
    <!-- Nav content -->
  </nav>

  <script>
    document.addEventListener("DOMContentLoaded", function() {
      const navbar = document.querySelector("nav");
      const navStyle = "border: 1px solid blue;";
      navbar.style.cssText = navStyle;
    });
  </script>
</body>
External Stylesheet
This is the most common way to handle complex designs. The biggest advantage of this method is that multiple pages can reuse the same styles, which provides the developer with consistency and follows the DRY (Don't Repeat Yourself) principle. External stylesheets can be added in the element of an HTML document like so: 
<head>
  <link rel="stylesheet" href="styles.css">
</head>
It's important to remember that more CSS is not always better. If CSS files get too big, it could hurt website performance. Adding CSS should be a mindful and intentional activity. If the developer cannot explain why they used a particular line of CSS, they should consider removing it.
